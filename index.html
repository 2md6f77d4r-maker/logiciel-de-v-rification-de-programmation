Manuel d’utilisation du logiciel

Ce logiciel sert à analyser automatiquement un fichier Excel de programmation. Il lit le planning, détecte les agents, les repos, les absences, les journées non couvertes, les anomalies réglementaires et propose des solutions possibles.

L’analyse se fait directement dans le navigateur. Le fichier Excel d’origine n’est pas modifié.

1. Démarrer le logiciel

  • Ouvre le fichier dans ton navigateur.
  • Le logiciel fonctionne hors ligne une fois ouvert, sauf pour les bibliothèques externes si elles ne sont pas déjà disponibles dans le cache du navigateur.
  • Le bouton 🌙 permet de passer en mode sombre. Le bouton ☀️ permet de revenir au mode clair.

2. Charger un fichier Excel

  • Clique sur “Fichier Excel”.
  • Sélectionne un fichier au format .xlsx, .xls ou .xlsm.
  • Le logiciel recherche en priorité la feuille “Modifications faite”.
  • Si cette feuille n’existe pas, la première feuille disponible est utilisée.
  • Tu peux changer la feuille analysée avec le menu “Feuille analysée”.

3. Sauvegarde automatique

  • Après chargement d’un fichier Excel, le planning est mémorisé automatiquement dans le navigateur.
  • Quand tu fermes puis rouvres le même fichier sur le même navigateur, le dernier planning se recharge automatiquement.
  • Si tu charges un nouveau fichier Excel, il remplace automatiquement l’ancien planning mémorisé.
  • Le bouton “Effacer le planning mémorisé” supprime la sauvegarde locale.

Important : cette sauvegarde reste liée au navigateur utilisé. Si tu ouvres le HTML sur un autre ordinateur ou un autre navigateur, le planning mémorisé ne suivra pas.

4. Structure Excel attendue

  • Une ligne doit contenir les dates du planning.
  • Les agents doivent être placés en lignes, avec leurs journées sous les dates.
  • La ligne située sous chaque agent peut contenir les temps de travail.
  • La zone TAS sert à identifier les journées non attribuées ou non couvertes.
  • Le logiciel peut lire une structure proche d’un planning classique, mais il dépend de la cohérence du fichier Excel.

5. Paramètres principaux

  • Codes repos : exemples R, RP, REPOS.
  • Codes absence : exemples CP, CA, FORMATION, MALADIE, ABS.
  • Seuil jours consécutifs : 5 par défaut.
  • Repos minimum par agent : 8 par défaut.
  • Minimum agents en service par date : utilisé pour contrôler la couverture.
  • Type de programmation : Paris ou Lyon.

6. Règles Paris / Lyon

  • Paris : la journée 400 doit être suivie de 401.
  • Paris : les journées 300/301 et 600/601 ne sont pas imposées en couple.
  • Lyon : la journée 300 doit être suivie de 301.
  • Lyon : la journée 600 doit être suivie de 601.
  • Ces règles sont utilisées dans les anomalies et dans les solutions proposées.

7. Tableau de bord

  • Affiche les indicateurs principaux du fichier analysé.
  • Présente des graphiques sur le travail, les repos, la couverture et les codes détectés.
  • Permet d’exporter un rapport complet en PDF.
  • Le bouton “Tout exporter” lance les exports disponibles.

8. Onglet Agents

  • Liste chaque agent détecté dans le fichier.
  • Affiche les jours travaillés, repos, absences, cases vides et heures totales.
  • Un clic sur un agent ouvre une fiche détaillée.
  • La fiche agent permet de visualiser la charge, les séquences et les points de vigilance.

9. Onglet Charge de travail

  • Analyse la fatigue potentielle de chaque agent.
  • Le score tient compte des heures, des repos, des absences et des séquences de travail.
  • La base de référence est 140 heures sur 28 jours.
  • Des couleurs permettent d’identifier rapidement les niveaux faible, modéré, élevé ou critique.
  • L’export PDF peut être filtré par agent et par plage de dates.

10. Onglet Anomalies

  • Affiche les problèmes détectés automatiquement.
  • Exemples : journée TAS non couverte, repos insuffisants, trop de jours consécutifs, journée liée manquante.
  • Les anomalies dépendent des paramètres sélectionnés en haut de page.
  • Après changement de paramètre, clique sur “Recalculer”.

11. Onglet Solutions possibles

  • Propose des agents pouvant couvrir une journée non attribuée sous TAS.
  • Le logiciel vérifie que l’agent est en repos sur la date concernée.
  • Il vérifie aussi que le minimum de repos reste respecté.
  • Il évite de créer une séquence de travail non conforme.
  • Les agents refusés sont listés avec la raison du refus.

12. Onglet Vue planning

  • Affiche le planning sous forme de tableau proche d’un Excel.
  • Les couleurs permettent de distinguer les services, repos, absences et journées liées.
  • Un clic sur une ligne agent permet d’ouvrir le détail de l’agent.
  • Le bouton flottant ▦ permet de revenir rapidement à la vue planning.

13. Onglet Calendrier

  • Affiche une vision par date.
  • Vert : journée correctement couverte.
  • Jaune : couverture faible selon les paramètres.
  • Rouge : au moins une journée non couverte ou problématique.

14. Exports disponibles

  • Agents : Excel et PDF.
  • Anomalies : Excel et PDF.
  • Solutions possibles : Excel et PDF.
  • Vue planning : Excel et PDF.
  • Charge de travail : Excel et PDF filtrable.
  • Rapport complet : PDF.

15. Boutons utiles

  • Recalculer : relance l’analyse avec les paramètres actuels.
  • Annuler le recalcul : revient à l’état précédent si disponible.
  • ↑ : remonte en haut de la page.
  • ▦ : ouvre rapidement la vue planning.
  • 💡 : ouvre ce manuel d’utilisation.

16. Limites importantes

  • Le logiciel dépend de la structure du fichier Excel chargé.
  • Un fichier très différent du modèle peut être mal interprété.
  • Le logiciel aide à détecter les erreurs, mais ne remplace pas une validation humaine.
  • La sauvegarde automatique ne modifie pas le fichier Excel source.
  • La sauvegarde automatique ne crée pas un nouveau fichier : elle garde les données dans le navigateur.

Détection automatique utilisée :

  • Recherche de la ligne contenant les dates Excel ou dates lisibles.
  • Les colonnes de planning sont les colonnes contenant une date.
  • Une ligne agent est retenue si elle contient un nom à gauche et des codes de service dans les colonnes datées.
  • La ligne située juste dessous est utilisée pour les temps de travail quand elle contient des durées numériques.
  • Les codes repos et absence sont modifiables dans les paramètres.
  • Une anomalie “plus de 5 jours” est déclenchée si une séquence de travail dépasse le seuil et n’est pas suivie de deux repos.
  • Le choix “Paris” autorise une journée 300 seule : aucun 301 n’est exigé derrière.
  • Le choix “Lyon” impose les enchaînements 300 → 301 et 600 → 601 dans les anomalies et dans les solutions proposées.
  • Les journées non couvertes sont recherchées uniquement sous la ligne “TAS”. Tout ce qui se trouve sous cette ligne est considéré comme non couvert jusqu’au prochain bloc de dates.
  • Les solutions possibles cherchent les agents en repos sur une journée non couverte. En mode Lyon, une proposition pour 300 inclut obligatoirement 301, et une proposition pour 600 inclut obligatoirement 601, sans enfreindre le minimum de repos ni la règle des jours consécutifs.

Le fichier ne modifie jamais ton Excel : l’analyse se fait uniquement dans le navigateur.
